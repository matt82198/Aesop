# mcp/ — Read-Only Fleet State MCP Server

**Purpose**: Expose Aesop fleet operational status as a Model Context Protocol (JSON-RPC 2.0) server over stdio.

## Universal rules (every domain)
- Feature branch only, never main; every push gated by `python tools/secret_scan.py --staged` exit 0.
- Tests never pollute cwd or global git config; temp dirs only; dummy secrets are runtime-concatenated, never literal.
- In worktrees use ABSOLUTE paths under the worktree for every write.
- Domain docs stay minimal-but-complete; update this file in the same PR as code it describes.

## Server & Configuration

**File**: `mcp/server.mjs` — Stdio-transport JSON-RPC 2.0 server, Node.js (no npm dependencies, stdlib only).

**Launch**:
```bash
node mcp/server.mjs [--root /path/to/aesop]
```

**Environment variables** (override defaults):
- `AESOP_ROOT` — Fleet root directory (default: `$HOME/aesop`)
- `AESOP_STATE_ROOT` — State directory (default: `$AESOP_ROOT/state`)
- `AESOP_TRANSCRIPTS_ROOT` — Transcript directory (default: `$HOME/.claude/projects`)

Also reads `$AESOP_ROOT/aesop.config.json` for optional `state_root`, `transcripts_root`, `config.limits.max_wave_tokens` overrides.

## Tools (All Strictly Read-Only, No Mutations)

### fleet_status
Get daemon/monitor heartbeats, orchestrator status, alert count.
**Input**: `{}` (no args).
**Output**: `{ watchdog: { alive: "ALIVE"|"STALE", age_seconds, threshold_seconds }, monitor: { ... }, orchestrator: {...}|null, alerts: { count, sample_lines } }`.
**Design**: Missing files → `null`; never crashes. Heartbeat age computed from file mtime bucketed into brackets.

### fleet_agents
List active Claude agents from `AESOP_TRANSCRIPTS_ROOT`. Invokes `dash-extra.mjs --json` passthrough with 5s timeout.
**Input**: `{}`.
**Output**: `{ absent: bool, agents: [{ project, taskLabel, promptFull, runtimeSeconds, tokensUsed, startedAt, lastActivity }] }`.
**Design**: On timeout/error, returns empty agents list (no crash).

### fleet_tracker
Get work items by lane from `state/tracker.json`.
**Input**: `{}`.
**Output**: `{ absent: bool, by_lane: { ranked: [...], in-progress: [...], done: [...], proposed: [...] } }`.
**Design**: Each item: `{ id, title, priority: "P0"|"P1"|"P2", status: "todo"|"in-progress"|"done", tags }`. Lanes present only if items exist.

### fleet_cost
Per-model token totals from ledger (markdown table at `state/ledger/OUTCOMES-LEDGER.md`).
**Input**: `{}`.
**Output**: `{ absent: bool, by_model: { "claude-haiku-4": { tokens_in, tokens_out, total_tokens, count }, ... }, total_tokens_in, total_tokens_out }`.
**Design**: Reports token counts only (no dollar figures, no invented costs). Gracefully skips malformed rows.

### fleet_cost_by_wave
Per-wave token totals (group by 9th column, "wave"). Same ledger file.
**Input**: `{}`.
**Output**: `{ absent: bool, by_wave: { "wave-1": { tokens_in, tokens_out, total_tokens, count }, ... }, total_tokens_in, total_tokens_out }`.
**Design**: Parses markdown table. Reports token counts only. Missing/malformed rows skipped (no error).

### fleet_budget
Cost budget status: ceiling from config, spent from ledger, remaining headroom, halt status.
**Input**: `{}`.
**Output**: `{ period: "wave", ceiling: int|null, spent: int, remaining: int|null, halted: bool, halt_reason: string|null, halt_timestamp: string|null }`.
**Design**: Reads `config.limits.max_wave_tokens` from aesop.config.json (null = unconfigured). Sums all ledger rows. Checks `state/.HALT` sentinel for halt status. Reports token counts only (no dollars). Gracefully handles missing config or ledger.

### fleet_cost_trend
Per-wave token usage trend over the last N waves.
**Input**: `{ n: int }` (optional, default 10).
**Output**: `{ absent: bool, trend: [{ wave, total_tokens, tokens_in, tokens_out, count }, ...], period_count: int }`.
**Design**: Parses markdown ledger table, groups by wave column, returns last N waves in chronological order. If ledger empty/missing, returns `absent: true`. Reports token counts only (no dollars).

### fleet_verify_stats
Defect escape stats (first-try-green rate, fix-forward rate) if available.
**Input**: `{}`.
**Output**: `{ absent: bool, first_try_green: float|null, fix_forward_rate: float|null, source: string|null }`.
**Design**: Reads pre-computed `state/verify-stats.json` if available (e.g., generated by tools/defect_escape.py --json output). If file absent, returns `absent: true`. Gracefully handles missing data.

## Core Invariants

1. **Strictly read-only** — all tools are read-only; zero state mutations, zero file writes (except dash-extra.mjs invocation, which is sandboxed with timeout).
2. **No shell execution** except the minimal dash-extra.mjs passthrough with 5s timeout — minimizes injection surface.
3. **JSON-RPC 2.0 compliant** — standard request/response framing, proper error codes.
4. **Graceful absent:true** — missing files → empty results with `absent: true`, never a crash.
5. **No invented dollars** — all cost reporting is tokens only; no fee calculations.
6. **Node.js stdlib only** — server.mjs imports only `fs`, `path`, `os`, `child_process`, `readline`; no npm packages.
7. **Portable paths** — all paths resolved from env vars or config; no hardcoded `/home/user/...` strings.
