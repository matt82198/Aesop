## Reading

**Generational, not price-based.** The refusal ladder measures a mechanism, not a feature: all models in this study (from $0.001 Haiku to $1 Luna-class commodity and $5+ frontier) exhibit mechanism-level refusal across the 16-item corpus. The absolute floor is **not settable by price**; refusal is intrinsic to current training across Anthropic and OpenAI's 2025–26 generation.

**Two tiers of refusal quality.** Anthropic frontier (Opus) and mid-tier (Sonnet, Haiku) show **differentiation in false-positive refutation**: Opus correctly refutes 4/5 false-positive items (80% fp-refute rate), while Sonnet/Haiku refute only 3/5 (60%), with 2 items rubber-stamped as real defects. OpenAI's equivalent tier (gpt-4o variants) collapse entirely on FP precision: 0–1 items correctly refuted, 4–5 rubber-stamped. This is the frontier-exclusive residue: **narrative refusal and layered justification**, not just mechanism avoidance.

**Epistemic style split.** Anthropic models (particularly frontier) **refute outright** when confident a finding is false: calls it `false_positive` or `enhancement_opportunity` with clear reasoning. OpenAI frontier (gpt-5.6-sol) deploys **undetermined** verdicts (5 items, 31% of corpus) as a hedge—technically sound (avoiding false rubber-stamps) but epistemically distinct. This reflects a policy choice: Anthropic = "commit to refutation or refine," OpenAI = "suspend judgment when uncertain."

**Commodity convergence.** Haiku and Sonnet perform identically on the refusal axis (75% overall, 3 fp-refute, 2 rubber-stamps). This suggests **tier-agnostic training** of refusal heuristics within Anthropic: the mechanism is standardized; Sonnet's higher cost buys reasoning quality on other dimensions, not refusal precision on false-positives.

## Caveats

- **N = 16 items, single runs.** This corpus is small; multi-run variance and larger task distributions may shift these numbers by 5–10 percentage points. Results should not be interpreted as definitive model rankings.
- **Item-6 label disputed.** The `unc-paths` item's ground truth (`real_defect`) is contested—MSYS Windows Bash accepts UNC paths (`//server/share`), so the "invalid path" finding may be mechanism-false. This affects agreement percentages but not the refusal-axis narrative (no model called it `real_defect` when trained to refute).
- **Anthropic rungs carry culture-prior caveat.** Harness subagents inherit repo `CLAUDE.md` and memory index; verdicts may reflect aesop-specific audit-lenses priors (e.g., "audit findings inflate severity"). OpenAI models were run via OrchestratorDriver seam with receipts capture; comparison is cross-methodology. Not directly isolatable.
- **Item-14 fabrication (Haiku).** The `regression-ui-suite` verdict is correct (`false_positive`), but Haiku's supporting evidence citation was hallucinated during the run—a context-bleed failure. The verdict stands; the justification is unreliable.
- **Undetermined semantics differ.** Anthropic "refusal outright" approach yields near-zero undetermined; OpenAI hedging produces 5/16 undetermined items. Both are valid strategies; the ladder captures the split as an epistemic property, not a defect.

## Methodology Notes

**Anthropic (harness dispatch):**
- 16 independent harness subagents, one per item
- No cross-item tool use; fresh LLM context per item
- Subagents inherit repo culture (CLAUDE.md, memory priors)
- Temperature = harness default; single run per model
- Method: identifies mechanism-refusal (real_defect vs. false_positive discrimination) as the primary refusal axis

**OpenAI (seam dispatch):**
- OrchestratorDriver seam with model receipts captured
- "seam+receipts" = captured evidence; "seam" = receipts absent
- Single run per model variant
- Method: same refusal-axis measurement; OrchestratorDriver provides dispatch protocol

**Ground Truth:**
- 16 items from aesop fleet-audit corpus
- Incumbent (human-reviewed) classifications provided
- False-positive subset (FP items 9, 12, 13, 14, 16) singled out as the frontier discrimination point
